Threat 1- User Credentials Disclosed:
Likelihood: Medium. An attacker would need access to netwrok traffic, stored data, or the client device to obtain user credentials. While modern systems often use security predoctions,weak storage or unencrypted communication could expose passwords.
Impact: High. If an attacker obtains valid user credentials, they can autheniticate as a legitimate user and gain unauthorized access to the incubator system. This would allow them to view sensitive information or issue commands.
Risk: A medium likelihood combined with a High impact results in a Medium risk. We agree with this rating because credetial theft requires some attacker capability, but the consequences of a successful attack are significant.

Threat 2- Authentication Token Capture:
Likelihood: High. If authentication takens are transmitted or stored in plaintext, an attacker monitoring network traffic or compromising a client device could capture them without needing the user's password.
Impact: Critical. A stolen authentication token allows an attacker to impersonate a legitimate user immediately. This could provide unrestricted access to system functions and potentially allow malicious changes to incubator settings.
Risk: A high likelihood and critical impact produce a critical risk. We agree because plaintext athentication tokens are relatively easy to exploit once exposed and provide direct access to the system.

Risk: A low likeihood and critical impact correspond to a medium risk. We agree with this because although physical attacks are less common, the conseqiences are severe enough that mitigation plans should still exist.

Threat 4- Distributed Denial-of-Service (DDoS):

Impact: High. A successful DDoS attack prevents legitimate users from accessing the system, interrupting monitoring and management of the incubator.
Risk: A medium likelihood and High impact produce a Medium risk. We agree because service disruptions are realistic threats, although they generally do not compromise data directly.
