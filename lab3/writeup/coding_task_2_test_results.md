## Platform limitations and known issues

- The interactive Tk/Matplotlib simulator GUI was not manually opened in this
  noninteractive validation. Its direct-execution startup path was compiled,
  while side-effect-free import and the actual secure UDP server/client path were
  tested.
- A complete browser/database login was not automated because the database and
  Coding Task 1 credential challenge are deliberately out of scope. Flask
  startup and security configuration were validated, and UDP behavior was
  exercised end to end.
- The full historical `requirements.txt` scientific stack was not installed on
  Python 3.13. Only the pinned packages needed for Coding Task 2 tests and Flask
  startup were installed locally. Normal users should install the requirements
  in a compatible virtual environment as documented in the README.
- UDP is intentionally not retried automatically. Packet loss produces a client
  timeout; retries would require replay-aware response caching.
- The bounded replay cache rejects new requests if 1,024 still-live request IDs
  are already present. This preserves replay evidence and prevents memory
  growth, but a client possessing the transport key could cause temporary
  availability loss until entries expire.
- Production browser access must use HTTPS and set
  `INCUBATOR_COOKIE_SECURE=1`. The documented default workflow binds Flask to
  loopback for local development.
